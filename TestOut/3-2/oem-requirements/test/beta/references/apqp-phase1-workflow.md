## 总览

APQP 阶段1（报价准备）的目标：收到 OEM 技术文件包后，完成全部报价前准备工作并生成 TDR（Technical Documents Required）文件包。

**时间节点**：RFQ 收到 → TKO（Technical Kick-Off）前提交 TDR。

**核心输入**：
- OEM 技术文件包（PDF/DOCX/XLSX/PPTX 混合）
- 公司能力档案（company-profile/，可选）

**核心输出**：
- out/ 目录下的全套 TDR 文件 + 辅助分析文件

## TDR 交付物清单

以 Stellantis 为例（其他 OEM 有类似但不完全相同的要求）：

| TDR # | 文件 | 说明 | AI可自动程度 |
|:---:|------|------|:---:|
| 1 | 成本分解 | BOM + 工艺成本 + 管理费 | 80%（需确认费率） |
| 2 | 虚拟分析 | CAE 仿真需求评估 | 仅判断适用性 |
| 3 | CDS 开发表 | 里程碑 + 时间计划 | 90%（模板化） |
| 4 | 3D 模型 | CAD 工程师手工完成 | 0%（标阻塞） |
| 5 | SDT 团队表 | 项目团队配置 | 100%（读 company-profile） |
| 6 | 例外事项清单 | 客户要求 vs 我方能力 | 85%（有能力声明时） |
| 7 | RASI 矩阵 | 职责分配 | 95%（模板化） |
| 8 | 补充材料清单 | 测试样件 + 费用 | 80%（需确认单价） |

**辅助文件**（非 TDR 编号，但报价必需）：

| 文件 | 说明 |
|------|------|
| A5 特殊特性清单 | CC/SC 特性及验证方法 |
| A6 质量目标表 | PPM/可靠性/Cpk/清洁度目标 |
| 缺失项跟踪表 | 引用但未提供的文档清单 |
| README 分析报告 | 文件包结构化分析 |

## Step 1 文件解析细节

### 文件类型识别与读取

| 文件类型 | 典型内容 | 读取方法 |
|---------|---------|---------|
| PDF | 性能规范（PF.xxxxx）、通用规格 | pdftext → 扫描件用 marker → 兜底 Read tool |
| DOCX | CTS（零部件技术规范）、交付物说明 | python-docx: paragraphs + tables |
| XLSX | SSTS（采购技术要求）、交付物清单、BOM | pandas + openpyxl |
| PPTX | 零件描述、项目概述 | python-pptx |

### OEM 识别规则

从文档内容推断 OEM：

| 线索 | OEM |
|------|-----|
| SSTS、PF.xxxxx、CTS-xxxxx、Stellantis 字样 | Stellantis |
| Lastenheft、TL-xxxxx、VW/Audi 字样 | VW Group |
| SDS、TSH-xxxxx、Toyota/Lexus 字样 | Toyota |
| GS xxxxx、BMW/Mini 字样 | BMW |

不确定时问用户。

### 文件角色分类

1. **采购技术总要求**：SSTS（Stellantis）、Lastenheft（VW）— 最高级别，定义总体要求
2. **零部件技术规范**：CTS — 针对具体零件的详细技术要求
3. **性能标准/规范**：PF.xxxxx — 某类零件的通用性能测试标准
4. **零件描述/BOM**：PPT/图纸 — 结构、材料、装配关系
5. **交付物清单**：要求供应商提交的文件列表
6. **报价文件清单**：TDR 具体格式要求

### 结构化数据提取清单

- [ ] 零件号、零件名、车型、发动机/平台
- [ ] BOM（子零件 / 材料 / 数量 / 重量）
- [ ] 性能指标表（爆破 / 泄漏 / 拉脱力 / 清洁度 / 耐久等）
- [ ] 测试矩阵（测试项 / 样件数 / 验收标准 / DV/PV 区分）
- [ ] 引用规范清单（编号 + 名称 + 是否已有）
- [ ] 项目时间约束（SOP 日期、TKO 日期等）
- [ ] 签署人 / 联系人

## Step 2 完整性检查细节

### 紧急度分级标准

| 级别 | 含义 | 颜色 | 示例 |
|:---:|------|:---:|------|
| 最高 | 阻塞报价 | 红 | 3D CAD、核心性能规范（PF.90197）、CTS |
| 高 | TKO 前需要 | 橙 | 验证计划模板、PPAP 等级确认、包装规范 |
| 中 | DV 前到位即可 | 黄 | SAE/ISO 行业标准、材料标准 |
| 低 | 可后续确认 | 绿 | 适用性存疑的文件、参考性文件 |

### 来源分类

- **向客户要**：CTS、PF 规范、3D 模型、PPAP 等级
- **自行购买**：SAE 标准、ISO 标准
- **内部收集**：公司能力数据、费率、团队配置

### 输出格式

`缺失项跟踪表.xlsx` 含：
- 缺失文档名、引用来源、紧急度、来源方、处理状态（下拉选单）
- 状态选项：未发起 / 已发起 / 已收到 / 不适用
- 自动统计公式

## Step 3 特性提取细节

### CC/SC 分类标准

| 类别 | 全称 | 判定条件 | 示例 |
|:---:|------|---------|------|
| CC | Critical Characteristic | 影响安全 / 法规 / 政府强制 | 爆破压力、阻燃性、有害物质 |
| SC | Significant Characteristic | 影响功能 / 耐久 / 客户满意 | 泄漏率、脉冲耐久、清洁度 |

### 质量目标类型

| 类型 | 典型指标 |
|------|---------|
| 市场质量 | 12MIS 故障率、PPM |
| 可靠性 | P99C90 寿命、R95C90 无故障概率 |
| 过程能力 | Cpk ≥ 1.67（CC）、Cpk ≥ 1.33（SC） |
| 清洁度 | 颗粒数 / 残留物重量（ISO 16232） |
| 安全法规 | ECE R67、FMVSS 等 |
| 环保 | IMDS 登记、GADSL 合规 |

## Step 4 可行性评估细节

### 能力对比维度

| 维度 | 对比项 |
|------|--------|
| 制造工艺 | 注塑/弯管/焊接/表面处理 vs 客户要求 |
| 材料 | 可获取材料 vs 客户指定材料 |
| 测试能力 | 实验室设备 vs 测试矩阵要求 |
| 软件工具 | CAD/CAE/PLM vs 客户要求格式 |
| 产能 | 现有产能 vs 项目需求量 |
| 质量体系 | 现有认证 vs 客户要求认证 |

### 判定结果

| 判定 | 含义 | 处理 |
|:---:|------|------|
| ✅ 符合 | 完全满足 | 无需行动 |
| ❌ 例外 | 不满足 | 建议替代方案（委外/格式转换/设备采购） |
| 🔵 待确认 | 信息不足 | 列入缺失项跟踪 |

## Step 5 TDR生成细节

### TDR1 成本分解结构

```
原材料成本
├── 主体材料（按 BOM 逐项：重量 × 单价）
├── 辅料（密封圈/弹簧/紧固件）
└── 小计 + 废料率

加工成本
├── 注塑（时间 × 费率）
├── 弯管 / 焊接 / 装配 / 检测...
└── 小计

其他费用
├── 模具分摊
├── 检具/工装
├── 管理费（加工成本 × 管理费率%）
├── 物流
└── 利润

= 单件总成本
```

### TDR3 CDS 里程碑

典型里程碑序列：RFQ → TKO → 模具设计 → 模具制造 → 首件 → DFMEA → PFMEA → DV 样件 → DV 测试 → 工装验证 → PV 样件 → PV 测试 → PPAP → Run@Rate → SOP

### TDR7 RASI 角色说明

| 字母 | 含义 |
|:---:|------|
| R | Responsible — 负责执行 |
| A | Accountable — 最终决策 |
| S | Support — 支持协助 |
| I | Informed — 知情通报 |

## Step 6 最终检查细节

### 一致性检查清单

1. **数字一致性**：A5 特性值 = TDR6 引用值 = TDR8 验收标准
2. **特性覆盖**：A5 每项 CC/SC 在 TDR6 都有对应行
3. **人员一致**：TDR5 人名 = TDR7 RASI 列头
4. **测试覆盖**：TDR8 测试项覆盖 A5 所有需验证的特性
5. **BOM 一致**：TDR1 BOM = README 中提取的 BOM
6. **缺失项状态**：所有"最高"优先级缺失项已有处理方案

## OEM 文档体系参考

### Stellantis

```
SSTS（采购技术总要求）
├── CTS（零部件技术规范）— 针对具体零件
├── PF.xxxxx（性能规范）— 针对零件类型的通用测试标准
├── CS.xxxxx（公司标准）— 内部流程/材料/方法标准
└── PRO.xxxxx（流程规范）— 包装/物流/变更管理
```

- SSTS 定义总框架 + 引用下级文件
- CTS 填具体值（覆盖 SSTS 通用值）
- PF 定义怎么测（测试方法/样件数/判定标准）

### VW Group

```
Lastenheft（技术需求书）
├── TL（技术交付条件）
├── PV（试验规程）
└── VW 标准号
```

### 通用结构

无论哪个 OEM，都有类似层级：
1. 采购总要求（框架）
2. 零件技术规范（具体值）
3. 性能/测试标准（怎么验证）
4. 引用标准（行业/公司内部）

## 常见零件类型参考

### 燃油管路 (Fuel Line)

**BOM 典型结构**：
- 主管体（PA 注塑件/钢管）
- 快插接头（QC）
- O-Ring / 密封圈
- 卡扣 / 固定夹
- 保护套管

**核心测试项**：
- 爆破压力（Burst Pressure）
- 泄漏测试（Leak Test, VLD）
- 脉冲耐久（Impulse Endurance）
- 拉脱力（Pull-off Force）
- 清洁度（Cleanliness, ISO 16232）
- 燃油兼容性（Fuel Compatibility）
- 振动耐久（Vibration Endurance）
- 温度循环（Thermal Cycling）

### 通用汽车零部件

**常见特殊特性**：
- 尺寸精度（CC/SC 视功能影响）
- 材料性能（拉伸强度/硬度）
- 表面处理（镀层厚度/粗糙度）
- 装配扭矩/力
- 密封性能
- 耐久性能
