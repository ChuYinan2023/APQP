CHAPTER 4 VALIDATION REQUIREMENTS:
================================================================================

Mandatory Deliverables:
  1. ONE Design Validation (DV) Plan per component
     Format: STELLANTIS norm 07740 annex 2
     Reference: Annex A - PF.90197

  2. ONE Production Validation (PV / AQF) Plan per component
     Format: STELLANTIS norm 07740 annex 2
     Reference: Annex A - PF.90197

Testing Guidelines:
  • All tests per STELLANTIS specifications must be listed
  • All tests must be present in plan (even if not executed)
  • If not performing test in DV: write reason in NOTE column, color line grey
  • If extending test from DV to PV: document in NOTE column, color line grey
  • Requalification tests in PV: Mandatory per STELLANTIS SPC 07740
  • Complement components (ICV, etc.): Must provide DV and PV plans

Standards for Validation:
  • Qualification procedure: STELLANTIS 07740
  • Format reference: STELLANTIS 07740 annex 2

SPECIAL REQUIREMENTS:
================================================================================

MATERIAL REQUIREMENTS
  • All materials must comply with STELLANTIS CS-9003 (MANDATORY)
  • Materials should be recyclable as much as possible
  • Recyclability calculated per STELLANTIS 00256 (green list)
  • Material alternatives evaluated by STELLANTIS during sourcing
  • Supplier must use IMDS system for material recyclability
  • Production approval ONLY after STELLANTIS receives material data via IMDS
  • Flagged criticalities must be reported to Material dept

CAD AND DESIGN REQUIREMENTS
  • CAD drawing standard compliance with §2.4.3 requirements
  • Latest UG/Teamcenter version (STELLANTIS standard)
  • All geometry changes per STELLANTIS Package Specification PRO.00004
  • Supplier must maintain STELLANTIS Package Team Requirements alignment
  • Self-certification of CAD parts quality per spec 07247

PACKAGE AND CONFIGURATION
  • Must comply with STELLANTIS Package Specification PRO.00004
  • All changes to CAD model must be fully documented
  • Supplier must share Package Team Requirements through all development phases
