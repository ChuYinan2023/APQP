阶段1 逐步执行流程（8 步 → TDR 8 项）

阶段1 的本质：**收到客户文件 → 理解 → 评估 → 交出 TDR 8 项报价文件**。

**步骤1：解析客户文件包**

|              | 内容                                       |
| ------------ | ---------------------------------------- |
| **输入**       | SSTS Excel、CTS docx、PF.90197 PDF、KP1 PPT |
| **AI 做什么**   | 读取全部文件，提取：技术要求表、BOM、性能指标、测试矩阵、引用规范清单     |
| **人做什么**     | 确认文件收齐，检查 AI 提取结果有无遗漏                    |
| **输出**       | 结构化技术要求表、BOM 表、性能指标表、引用规范清单              |
| **与 TDR 关系** | 不直接对应某项 TDR，但它是**所有后续步骤的基础**             |
| **状态**       | **已完成**                                  |

**步骤2：文件完整性审核 → 催要**

|              | 内容                                                         |
| ------------ | ---------------------------------------------------------- |
| **输入**       | 步骤1 的引用规范清单 + 手头全部文件                                       |
| **AI 做什么**   | 逐条核对：CTS 引用了 PF.90298 → 有没有？没有 → 标缺失；按阻塞/高/中/低分级           |
| **人做什么**     | 确认优先级，决定哪些必须催、哪些可以先跳过                                      |
| **输出**       | ① 缺失项清单（17项） ② 中英文催要邮件草稿                                   |
| **与 TDR 关系** | 不直接对应 TDR，但**解除阻塞**——3D CAD 和 PF.90298 不到手，TDR #1 和 #8 做不了 |
| **状态**       | 清单 **已完成**，邮件 **随时可执行**                                    |

**步骤3：提取特殊特性 + 质量目标**

|              | 内容                                                                                                        |
| ------------ | --------------------------------------------------------------------------------------------------------- |
| **输入**       | CTS、PF.90197、SSTS                                                                                         |
| **AI 做什么**   | ① 从 CTS/PF.90197 找出所有 CC（关键特性）和 SC（重要特性）标注项 ② 从 SSTS/PF.90197 提取 PPM 目标、Cpk 要求、可靠性指标（P99C90/R95C90）、清洁度目标 |
| **人做什么**     | 审核清单，补充供应商经验中知道但文件未标注的特性项；确认质量目标可达性                                                                       |
| **输出**       | ① 初始特殊特性清单 Excel ② 质量目标表                                                                                  |
| **与 TDR 关系** | 不直接对应 TDR，但特殊特性清单是后续 DFMEA/控制计划的种子，质量目标是 **TDR #6 例外事项** 判定的基准                                            |
| **状态**       | **可执行**                                                                                                   |

**步骤4：整理我方能力声明**

|              | 内容                                               |
| ------------ | ------------------------------------------------ |
| **输入**       | 公司内部资料：设备清单、IATF 16949 审核报告、测试设备台账、公司手册          |
| **AI 做什么**   | 提供标准模板框架（制造能力/测试能力/软件工具/产能 4 大类），辅助从已有文档中提取填充    |
| **人做什么**     | 逐项填入真实数据：有什么设备、能做什么测试、用什么 CAD、产能多少               |
| **输出**       | 我方能力声明文件                                         |
| **与 TDR 关系** | 不直接对应 TDR，但它是 **步骤5（TDR #6）的必要输入**——没有它，例外事项无法判定 |
| **状态**       | **待执行**（需内部收集）                                   |

**步骤5：例外事项清单 → TDR #6**

|              | 内容                                              |
| ------------ | ----------------------------------------------- |
| **输入**       | 步骤4 能力声明 + CTS 全部技术要求 + PF.90197 全部性能要求         |
| **AI 做什么**   | 逐条对比：客户要求 X → 我方能力 Y → 判定"符合/例外/待确认"；对例外项建议替代方案 |
| **人做什么**     | 审核每条判定结果，确认替代方案是否可行，补充商务层面的例外（如交期、产能）           |
| **输出**       | 例外事项清单 Excel：每行一条要求，列有判定结果 + 替代方案               |
| **与 TDR 关系** | **直接就是 TDR #6**                                 |
| **状态**       | **待执行**（被步骤4阻塞）                                 |

**步骤6：RASI 矩阵 + SDT → TDR #4 / #7**

|              | 内容                                                  |
| ------------ | --------------------------------------------------- |
| **输入**       | Deliverables 15 项交付物清单 + 我方项目团队人员名单（SDT）            |
| **AI 做什么**   | ① 按行业惯例预填 RASI（谁负责/谁批准/谁支持/谁知会） ② 生成 SDT 团队表模板      |
| **人做什么**     | 确定项目经理、设计工程师、质量工程师等人选；调整 RASI 分配                    |
| **输出**       | ① SDT 团队表 → **TDR #4** ② RASI 矩阵 Excel → **TDR #7** |
| **与 TDR 关系** | **直接就是 TDR #4 和 TDR #7**                            |
| **状态**       | **待执行**（需 SDT 人员名单）                                 |

**步骤7：补充材料清单 + 时间计划 → TDR #3 / #5 / #8**

|              | 内容                                                                                                    |
| ------------ | ----------------------------------------------------------------------------------------------------- |
| **输入**       | PF.90197 附录A 测试矩阵 + PF.90298（接头） + Deliverables + CDS 模板                                              |
| **AI 做什么**   | ① 从测试矩阵提取全部 DV/PV 测试项，自动计算样件数量（P99C90=15件、R95C90=45件） ② 基于 Deliverables 15 项生成甘特图框架 ③ 预填 CDS（组件开发表）模板 |
| **人做什么**     | ① 填入样件单价，得出测试总费用 ② 确认关键里程碑日期 ③ 确认 CDS 内容                                                              |
| **输出**       | ① 补充材料清单 Excel → **TDR #8** ② 时间计划/甘特图 → **TDR #3** ③ CDS 组件开发表 → **TDR #5**                          |
| **与 TDR 关系** | **直接就是 TDR #3、#5、#8**                                                                                 |
| **状态**       | **可部分执行**（PF.90298 未到手影响接头测试项；里程碑需客户确认）                                                               |

**步骤8：成本分解 + 2D 图纸 → TDR #1 / #2**

|              | 内容                                                           |
| ------------ | ------------------------------------------------------------ |
| **输入**       | BOM + **3D CAD 数据** + 内部成本数据（材料单价/工时费率/模具报价）+ 步骤7 的测试费用      |
| **AI 做什么**   | ① 从 BOM + 3D 数据推导工艺路线，生成 Stellantis 成本分解模板框架 ② 自动填充材料用量、工序清单 |
| **人做什么**     | ① 填入真实单价和费率 ② 出 2D 工程图纸（需 CAD 软件，AI 无法替代）                    |
| **输出**       | ① 成本分解 Excel → **TDR #1** ② 2D 工程图纸 → **TDR #2**             |
| **与 TDR 关系** | **直接就是 TDR #1 和 TDR #2**                                     |
| **状态**       | **被阻塞**（3D CAD 数据未到手）                                        |

**8 步 → TDR 8 项映射总览**

```
步骤1 解析文件        ─── 基础（不对应 TDR，但所有步骤依赖它）
步骤2 完整性审核+催要  ─── 解除阻塞
步骤3 特性+质量目标    ─── 种子数据
步骤4 能力声明        ─── 步骤5 的输入
步骤5 例外事项清单    ─── → TDR #6
步骤6 RASI + SDT     ─── → TDR #4 + TDR #7
步骤7 材料+计划+CDS   ─── → TDR #3 + TDR #5 + TDR #8
步骤8 成本+图纸       ─── → TDR #1 + TDR #2
```
