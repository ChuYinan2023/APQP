### PF.90068 和 PF.90002 的适用性存疑

这两份 PDF 都是 **DEF（尿素）系统**的规范，而 KP1 的零件是 **柴油滤清器至发动机的燃油供油管**（Fuel line diesel filter to engine）。这是两个不同的系统：

- DEF 系统：尿素溶液，用于尾气后处理（SCR）
- 燃油系统：柴油/汽油，用于发动机供油

PPTX 中明确写的是"所有零件须符合 **PF.90197**"，并未提及 PF.90068 和 PF.90002。这两份可能是客户一并发来的**参考文件**，但不一定是这个零件的直接采购要求。

## PPAP Level 3 — 18 项提交要求分析

### 术语说明

**PPAP**（Production Part Approval Process，生产件批准程序）是汽车行业供应商向 OEM 提交的一整套文件包，用于证明供应商有能力按照设计要求稳定地批量生产合格零件。

**PPAP 等级**（Submission Level）共分 5 级：

| 等级 | 要求 | 说明 |
|------|------|------|
| Level 1 | 仅提交 PSW | 最简单，文件供应商自行保留 |
| Level 2 | PSW + 有限支持文件 + 样件 | 交部分关键文件 |
| **Level 3** | **PSW + 全套支持文件 + 样件** | **默认等级，最常用，18项全交** |
| Level 4 | PSW + 客户指定文件 | 客户挑选需要的项目 |
| Level 5 | PSW + 全套文件现场备查 | 客户可随时去现场审核 |

> 不管哪个等级，供应商都必须完成全部 18 项工作，区别只是交多少给客户。

## 关键术语速查

### DVP&R — Design Verification Plan & Report（设计验证计划与报告）

APQP 流程中的核心文档，用于：
- **Plan（计划）**：列出所有需要执行的设计验证测试项目，包括测试条件、样本数量、接受标准
- **Report（报告）**：记录实际测试结果，判定 Pass/Fail

本项目中 PPT 第2页提到 "DV test is mandatory"，即供应商如果修改了卡扣设计，必须按 DVP&R 完成设计验证。

典型 DVP&R 包含的列：

| 列 | 内容 |
|---|------|
| Test Item | 测试项目（如爆破压力、脉冲耐久、盐雾腐蚀等） |
| Specification | 引用的规范/标准 |
| Acceptance Criteria | 接受标准 |
| Sample Size | 样本数量 |
| Test Condition | 测试条件（温度、压力、循环次数等） |
| Responsibility | 负责方（供应商/OEM） |
| Status / Result | 实际结果及判定 |

### DFMEA — Design Failure Mode and Effects Analysis（设计失效模式与影响分析）

在零件设计阶段系统性地识别"设计上可能出什么问题"。

以本项目阻尼器为例：

| 列 | 示例 |
|---|------|
| 功能 | 抑制燃油压力脉动，工作压力 4.5 bar |
| 潜在失效模式 | 膜片破裂 |
| 潜在影响 | 燃油泄漏 → 发动机熄火 → 安全风险 |
| 严重度 S | 9（涉及安全） |
| 潜在原因 | 膜片材料（54U6002+AgN91S）耐疲劳性不足 |
| 发生度 O | 4 |
| 现有设计控制（预防） | 按 PF.90197 选材 |
| 现有设计控制（探测） | DVP&R 脉冲耐久测试 |
| 探测度 D | 3 |
| RPN | 9×4×3 = 108 |
| 建议措施 | 增加脉冲循环次数验证 |

核心逻辑：**S × O × D = RPN**（风险优先数），RPN 高的项优先改进。

### MSA — Measurement System Analysis（测量系统分析）

验证"量具测得准不准"。最常用方法是 GR&R（Gauge Repeatability & Reproducibility）：

| 概念 | 含义 | 举例 |
|------|------|------|
| Repeatability（重复性） | 同一人、同一量具、测同一零件，多次结果一致吗 | 张三用卡尺测阻尼器外径 φ44.6，测 3 次 |
| Reproducibility（再现性） | 不同人测同一零件，结果一致吗 | 张三和李四各测 3 次 |

判定标准：

| GR&R % | 判定 |
|--------|------|
| < 10% | 合格 |
| 10% ~ 30% | 有条件接受，需改进 |
| > 30% | 不合格，量具系统不可信 |

### 三者关系

```
正式图纸（关键尺寸定义）
    │
    ├──→ DFMEA：这些设计特征可能怎么失效？
    │         │
    │         └──→ 识别出关键特性（CC/SC）
    │                    │
    └──→ MSA：测这些关键尺寸的量具可靠吗？
                         │
                         └──→ 尺寸报告（PPAP #9）才有意义
```

DFMEA 告诉你"哪些特性最重要"，MSA 保证"测这些特性的工具靠谱"。两者都依赖正式工程图纸。

## APQP 项目阶段与当前定位

### 项目时间线

```
RFQ(询价) → 报价 → Technical Review(技术评审) → 定点 → TKO(开模) → DV → PV → SOP
     ↑                                                    ↑
  当前阶段                                            花钱的起点
```

**关键节点说明：**

- **TKO**（Tooling Kick-Off，模具启动）：Stellantis 批准供应商正式开始制造模具的节点。开模意味着大额投资（通常几十万到上百万），所以 TKO 前客户要确认设计已冻结、风险已识别。TKO 之后再改设计 = 改模 = 烧钱。
- **TDR**（Technical Documents Required，技术报价所需文件清单）：Stellantis 告诉供应商"你要报价的话，先把这些文件准备好交给我"。是报价阶段的"入场券"。

### 当前阶段（报价）需要准备的文件

当前处于 **报价阶段**，需要准备的是 TDR 列出的 8 类文件，而非 PPAP 18 项（PPAP 是定点后逐步完成的）。

| # | TDR 要求的报价文件 | 谁准备 | 现有材料能支撑？ | 缺口 |
|---|---|:---:|:---:|---|
| 1 | 协同设计成本分解 | 供应商 | 部分 | 需要 3D 数据估算工艺和材料成本 |
| 2 | 虚拟分析 | — | N/A | TDR 写明本项目不适用 |
| 3 | CDS（零部件开发表） | 供应商 | 可开始 | 基于 CTS 可以编制 |
| 4 | 3D 模型（UG/Parasolid） | 供应商 | **缺** | 客户须先提供初始 3D，供应商才能做设计提案 |
| 5 | SDT（供应商开发团队表） | 供应商 | 可开始 | 不依赖客户 |
| 6 | 例外事项清单 | 供应商 | 可开始 | 基于 CTS + PF.90197 识别不符合项 |
| 7 | RASI 责任矩阵 | 供应商 | 可开始 | 不依赖客户 |
| 8 | 补充材料清单（测试件费用） | 供应商 | 部分 | 需要 PF.90298 确认接头测试要求 |

### 什么是 QFD

**QFD**（Quality Function Deployment，质量功能展开）是客户将"客户需求"转化为"技术要求"的文件。在本项目中，客户发来的 SSTS + CTS + PF 规范 + PPT 就是 QFD 的输出物。

收到 QFD 后，供应商按 AIAG 标准的 **APQP 5 个阶段**推进：

```
APQP 阶段1       阶段2            阶段3            阶段4           阶段5
策划与定义 → 产品设计开发 → 过程设计开发 → 产品过程确认 → 反馈与持续改进
 Plan &       Product        Process        Product &      Feedback &
 Define       Design         Design         Process        Continuous
                                            Validation     Improvement

对应项目节点：
RFQ/报价 → Technical Review → 定点/TKO → DV → PV → PPAP提交 → SOP → 量产
  (现在)
```
