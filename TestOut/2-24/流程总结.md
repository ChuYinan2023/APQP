# APQP文档处理流程总结 — FILTER项目

**项目：** Smart Fuel Filter — S K0 / FamB 2.2L Mjet DIESEL
**输入：** 2份Spec PDF（通用Spec 44页 + CTS 14页）
**输出：** 3份Excel + 工作底稿 + 知识回写

---

## 主流程

```
阶段1 文档清点与确认
  ↓ 用户确认缺失/冲突处理方式
阶段2 提取L1工程特性（系统级）
  ↓ 用户确认L1完整性
阶段3 提取L2零部件特性
  ↓ 用户确认BOM分解合理性
阶段4 QFD质量屋矩阵
  ↓ 用户确认最终交付
知识回写
```

每个阶段输出后**必须等用户说OK才进入下一阶段**。

---

## 阶段1：文档清点与确认

### 做什么
1. 逐页读取所有PDF，**全文不删不改**存入 `_工作底稿.md`
2. 按 文档→章节→页码 结构化整理
3. 识别文档类型和层级关系（本项目：通用Spec是框架 + CTS是项目层，冲突时CTS优先）
4. 找出缺失文档、冲突项、术语表

### 重点
- **工作底稿是后续所有阶段的数据源**，阶段2/3/4不再回读PDF
- Stellantis双文档模式：通用Spec(PF.90150) + CTS。通用Spec中写"As per CTS"的参数必须在CTS中找具体值
- 缺失文档要评优先级（高=影响核心参数 / 中=部分影响 / 低=仅参考）
- 冲突项两个值都记录，让客户决定取舍

### 本项目结果
- 8项缺失文档（D1-D8）：最关键的是工程图纸、CS.00056/CS.00263(E/E环境)、CS.00244(EMC)
- 4项冲突（C1-C4）：过滤效率@4μm / 水分离效率 / 新品压降 / E/E环境规范编号
- 用户决策：不补文档，按现有内容继续

### 输出
- `_工作底稿.md` — 全文存档
- 对话中展示清单（本项目额外输出了 `阶段1_文档清点与确认.md` 方便客户离线填写）

---

## 阶段2：提取L1工程特性（系统级）

### 做什么
1. 从工作底稿**逐章节**提取系统级工程特性
2. 顺序编号，特性分类从Spec章节内容自然归纳
3. 标注类别：**C(关键)** / **A(一般)**
4. **多条件参数逐条分行**（多粒径/多工况/多电压/多温度）
5. 冲突项在数据行备注列标注 + 汇总到缺口Sheet

### 重点
- **类别标注带中文说明**：写 `C(关键)` / `A(一般)`，不要光写字母
- 过滤效率必须按粒径分行（4μm/5μm/10μm/15μm/20μm/30μm各一行）
- 压降按工况分行（标称流量 vs 通用流量 vs 低温）
- 传感器参数如果通用Spec标注"example"，需在备注中说明"正式值在Product Card"
- Pinout每个PIN单独一行，均为C类

### 本项目结果
- **104条L1特性**，21个自然分类
- **57条C类（关键）+ 47条A类（一般）**
- 9项缺口 + 4项冲突

### 输出
- `阶段2_L1工程特性清单.xlsx`
  - Sheet 1「工程特性清单」：ID | 特性分类 | 工程特性名称 | 目标值/要求 | 原文描述 | 类别 | 文件来源 | 章节 | 页码 | 备注
  - Sheet 2「缺口与冲突」
  - 样式：C类行粉红 #FFE6E6，冲突行黄色 #FFD966

### 使用模板脚本
从 `scripts/gen_l1_template.py` 复制到output，只改数据区（L1_DATA / GAP_DATA / CONFLICT_DATA），格式代码一字不改。

## 阶段3：提取L2零部件特性

### 做什么
1. 从工作底稿**推断BOM结构树**（从测试对象/接口描述/材料要求/设备分类反推）
2. 从L1逐条向下分解到零部件级
3. 每条L2关联对应的L1 ID
4. 标注缺口

### 重点
- BOM推断来源：
  - 设备分类表(Table 4)列出的独立电子设备 → 各自成为子零部件
  - 液压接口(进口/出口) → 液压快接
  - 密封测试(外部/内部/垫片) → 密封件
  - 壳体/滤芯/集水碗从测试对象描述中识别
- 同一个L1可能分解到多个L2（如L1"腐蚀试验"→壳体耐腐蚀+PTC耐腐蚀后功能+NTC耐腐蚀后功能+WiF耐腐蚀后功能）
- 缺口按优先级分：高(图纸/环境规范/EMC) > 中(传感器正式参数/快接规范) > 低(滤材规格/保护帽)

### 本项目结果
- **86条L2特性**，10个零部件/组件
- 12项缺口（4高/6中/2低）
- 零部件分布：滤芯18条 > PTC加热器13条 > WiF传感器12条 > 壳体10条

### 输出
- `阶段3_L2零部件特性清单.xlsx`
  - Sheet 1「零件特性清单」：ID | 零件/组件名称 | 零件特性 | 目标值/要求 | 对应L1 ID | 文件来源 | 章节 | 备注
  - Sheet 2「缺口与冲突」

## 阶段4：QFD质量屋矩阵

### 做什么
1. 构建L1(行) × L2(列) 关联矩阵
2. 标注关联强度：◎=9(强) / ○=3(中) / △=1(弱) / 空=无关联
3. 检查空行（L1无L2关联）和空列（L2无L1关联）
4. 识别关联密度最高的L2（设计关键件）

### 重点
- **关联强度判断**：
  - ◎(9)=直接对应（L2就是L1的分解）
  - ○(3)=间接影响（如环境耐久对多个零部件的影响）
  - △(1)=弱关联（如工作条件对结构件的基本约束）
- 工作条件(L1 #1-7)、环境耐久(L1 #71-77)、耐腐蚀(L1 #78-79)等系统级要求会关联到多个零部件
- 验证类要求(如P99C90/R95C90)是元要求，不直接关联零部件——空行是正常的
- 矩阵覆盖率通常很低（本项目2%），因为大矩阵天然稀疏

### 本项目结果
- **104 L1 × 86 L2 = 8,944格**
- **233个有效关联格**
- 空行L1：#96, #97（验证标准，元要求，正常）
- 空列L2：无
- **设计关键件Top5**：滤材规格(PC-28) > 壳体爆破(PC-03) > 壳体耐热循环(PC-09) > 电连接器(PC-67) > 密封件气密(PC-80)

### 输出
- `阶段4_QFD质量屋矩阵.xlsx`
  - Sheet 1「QFD矩阵」：蓝色标题区 + L2三行表头(零件名/ID+特性/目标值) + L1数据行，冻结D4
  - Sheet 2「审查」：空行/空列/Top5/覆盖率
