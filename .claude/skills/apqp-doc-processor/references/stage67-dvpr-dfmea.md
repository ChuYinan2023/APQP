# 阶段6-7：DVP&R / DFMEA（可选扩展）

## 阶段6：DVP&R框架

**触发条件**: 用户明确要求，或文档中包含Annex A/DV/PV表

基于L1/L2特性 + 测试标准 + 样本量信息，自动生成DVP&R草稿：

| 列 | 来源 |
|----|------|
| 测试项目 | L1工程特性名称 |
| 规范章节 | L1来源章节号 |
| 测试标准 | L1条件中的ISO/SAE标准 |
| 接受标准 | L1目标值 |
| DV样本量 | Annex A中的DV列 |
| PV样本量 | Annex A中的PV列 |
| 置信度 | P99C90 / R95C90 等 |
| 责任方 | 通常为Supplier |
| CC频次 | Annex A-2中的频次 |

## 阶段7：DFMEA框架

**触发条件**: 用户明确要求

基于L1→L2映射，自动生成DFMEA结构草稿：

```
功能（L1） → 失效模式（L1不满足） → 失效影响（系统级后果）
                                    → 失效原因（L2特性偏离）
                                    → 现行控制（测试/检验）
```

映射规则：
- L1的每个工程特性 → 一个或多个"功能"
- L1不满足 → "潜在失效模式"
- L1×L2强关联(◎) → "潜在失效原因"（L2偏离导致L1失败）
- 测试标准 → "现行设计控制"（预防/探测）
