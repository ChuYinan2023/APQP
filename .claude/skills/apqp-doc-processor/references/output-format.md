# 输出规范

## 目录结构

所有阶段输出统一写入 `{SPEC文件夹}/output/` 目录：

```
{SPEC文件夹}/output/
├── {文件名}_text.md                  ← 阶段0.5 中间文件（PDF原文）
├── 阶段1_文档关系分析.md
├── 阶段2_翻译解读与结构树.md
├── 阶段3_L1工程特性清单.xlsx        ← Excel
├── 阶段4_L2零部件特性清单.xlsx      ← Excel
├── 阶段5_QFD质量屋矩阵.xlsx        ← Excel
├── 阶段6_DVPR框架.xlsx             (可选, Excel)
└── 阶段7_DFMEA框架.xlsx            (可选, Excel)
```

## 格式规则

| 阶段 | 格式 | 原因 |
|------|------|------|
| 0.5 PDF提取（中间文件） | .md | 原文文本，供后续阶段读取 |
| 1 文档关系分析 | .md | 文字描述为主 |
| 2 翻译解读 | .md | 结构树+术语表，文字为主 |
| 3 L1工程特性 | **.xlsx** | 结构化表格，便于筛选/排序/交付 |
| 4 L2零部件特性 | **.xlsx** | 结构化表格，含多Sheet |
| 5 QFD质量屋 | **.xlsx** | 矩阵，Excel天然适合 |
| 6-7 DVP&R/DFMEA | **.xlsx** | 行业标准交付格式 |

## Excel生成规则

- **必须使用标准模板脚本**，禁止从零编写生成代码
- 模板路径：`.claude/skills/apqp-doc-processor/scripts/`
  - `gen_l1_template.py` → 阶段3 L1工程特性清单
  - `gen_l2_template.py` → 阶段4 L2零部件特性清单
  - `gen_qfd_template.py` → 阶段5 QFD质量屋矩阵

**使用方式**（每次生成时严格遵循）：
1. 将对应模板**复制**到 `output/gen_l1.py`（或 gen_l2.py / gen_qfd.py）
2. **只修改** `CONFIG`、`L1_DATA`/`L2_DATA`/`MATRIX` 等数据区（有注释标注）
3. 格式代码区（`def build_xxx` 函数）**一字不改**
4. 将 `CONFIG["输出路径"]` 改为相对于 output 目录的文件名
5. 运行：`python gen_l1.py`

- 每个Excel第一个Sheet为"封面"，包含元数据：

| 字段 | 值 |
|------|---|
| 零部件名称 | {名称} |
| OEM | {OEM名称} |
| 来源文档 | {文件名列表} |
| 提取日期 | {YYYY-MM-DD} |
| 阶段 | {N} - {阶段名} |

- 表头行冻结、列宽自适应
- 关键列加粗或填充色区分

## 各阶段Excel Sheet结构

### 阶段3 — L1工程特性清单.xlsx

| Sheet | 内容 |
|-------|------|
| 封面 | 元数据 |
| 工程特性清单 | ID / 特性分类 / 工程特性名称 / 目标值/要求 / 文件名称 / 层级 / 页码 / 章节 / 图表 / 原文描述 / 类别 / 冲突项ID / 冲突/差异说明 / 分析建议 / 客户裁决 |
| CC持续符合性 | **（可选）** 从Spec Annex中提取的持续符合性(Continuing Conformance)要求：测试项 / 样本量 / 接收准则 / 频次(100%/每日/每周/每季) |
| 反向校验 | 章节 / 覆盖状态 / 备注 |
| 统计汇总 | 按类别(C/A)统计数量 |

### 阶段4 — L2零部件特性清单.xlsx

| Sheet | 内容 |
|-------|------|
| 封面 | 元数据 |
| BOM结构 | 零部件编号 / 名称 / 类型 / 父级 |
| 零件特性清单 | ID / 零件/组件名称 / 零件特性 / 目标值/要求 / 对应第一阶段工程特性 / 文件来源 / 章节 / 备注 |
| 缺口清单 | GAP编号 / 缺失信息 / 影响L2 / 关联L1 / 所需文档 / 优先级 |
| 反向校验 | 章节 / 覆盖状态 / 备注 |
| 统计汇总 | 各零部件特性数量 |

### 阶段5 — QFD质量屋矩阵.xlsx

| Sheet | 内容 |
|-------|------|
| 封面 | 元数据 |
| QFD矩阵 | 标题区(A1:C3合并, 蓝底白字)；L2列3行表头(Row1=零件名称/Row2=PC-ID+特性/Row3=目标值)；行=L1(ID+特性名称), 列=L2, 值=◎/○/△ |
| 屋顶矩阵 | L2×L2，值=+/- |
| 判定依据 | 关键关联的判定说明 |
| 全局审查 | 空行/空列检查、关键件、关键冲突 |

## 术语表管理

- 术语表保存到项目级 `{项目根目录}/术语表.md`，可跨零件复用
- 后续处理同项目其他零件时，先加载已有术语表，仅增量更新
- 格式：`| 英文 | 中文 | 备注/上下文 |`
