# 阶段1：多文档关系分析

当处理2个以上文档时，必须先分析文档间关系。

## 文档层级识别

| 层级 | 文档类型 | 识别特征 | 优先级 |
|------|---------|---------|--------|
| L0 | 法规/国际标准 | ISO/SAE/EN编号 | 最高（不可偏离） |
| L1 | OEM通用规格 | PF./CS./QR.编号、适用于多项目 | 高（通用基线） |
| L2 | OEM应用规格(CTS) | 特定项目号、特定车型 | 高（覆盖通用值） |
| L3 | 产品卡/接口文件 | Product Card、Pinout、EE Checklist | 中（具体参数） |
| L4 | 供应商规格/图纸 | 供应商名称+型号、DV报告 | 参考 |

## 覆盖规则

- CTS参数 **优先于** 通用Spec中的通用值
- 当CTS说"按CTS"时，以CTS具体数值为准
- 当CTS未提及某项时，以通用Spec为准
- **冲突标记**：两文档对同一参数给出不同值时，标注 `[冲突]` 并列出两个值

## 输出内容

- 文档关系图（层级树形）
- 覆盖规则表
- 冲突项列表
