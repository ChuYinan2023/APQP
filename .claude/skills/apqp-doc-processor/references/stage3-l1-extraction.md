# 阶段3：提取L1工程特性

## 分类体系

使用固化的7大分类，详见 [l1-classification.md](l1-classification.md)

## 提取步骤

1. 基于结构树逐章节提取工程特性（系统级）
2. 按7大分类归类，使用固定ID格式：`{分类}-{序号}`（如 F-01, P-02）
3. 每条记录包含：

| 字段 | 说明 |
|------|------|
| ID | 分类代码-序号 |
| 工程特性名称 | 中文描述 |
| 目标值/范围 | 具体数值或定性要求 |
| 单位 | 物理单位 |
| 条件 | 测试条件/标准/温度/流量等 |
| 来源 | 文档名 §章节号 |
| 文档层级 | 来自通用Spec还是CTS（标注覆盖关系） |

4. **反向比对**原文：逐章节检查遗漏、错分类、重复
5. 分类模糊项标注 `[待确认分类]` 并暂定主分类

## 合并规则

- 同一参数在通用Spec和CTS中都有时，**两个值都记录**，CTS值标为主值
- 多个粒径/多个工况/多个电压等级/多个温度段的参数，**必须逐条分行记录**（每行一个具体验收值），不可合并为一行。理由：合并后无法在DVP&R中逐项对应试验计划。
  - 例：清洁度颗粒分级 → 50-100μm / 100-200μm / >200μm 各一行
  - 例：加热器寿命 → 14V/5000次、18V/500次、24V/10次 各一行
  - 例：WiF自检时间 → -30~110°C段 和 -40~125°C段 各一行

## 输出内容

- 按分类排列的L1特性表
- 反向校验记录（逐章节覆盖确认）
- 统计汇总（各分类数量）
