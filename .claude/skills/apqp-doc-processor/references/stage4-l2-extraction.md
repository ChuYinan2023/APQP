# 阶段4：提取L2零部件特性

## BOM结构树推断

先建立BOM结构树，即使无正式BOM也基于文档内容推断：

1. **从测试项目反推**：每个独立测试对象通常是一个子零部件
2. **从接口描述反推**：每个定义了接口的部分通常是一个子零部件
3. **从材料要求反推**：不同材料要求通常对应不同子零部件
4. **从设备分类表反推**：分类表中列出的设备通常各自独立

## 零部件分解

根据零件类型采用不同策略，详见 [component-decomposition.md](component-decomposition.md)

## 提取步骤

1. 建立BOM结构树
2. 从L1逐条向下分解到零部件级
3. 逐零部件提取特性，按 `L2-{零部件序号}.{特性序号}` 编码
   - **分行规则同L1**：多粒径/多工况/多电压/多接口位置的参数必须逐条分行，不可合并
4. 每条L2记录包含：L2 ID、特性名称、目标值/范围、单位、关联L1 ID、来源
5. **反向比对**原文
6. **缺口分析**：标记无法从现有文档支撑的L2特性

## 缺口分析

详见 [gap-documents.md](gap-documents.md)

## 输出内容

- BOM结构树
- 各零部件特性表
- 反向校验记录
- 缺口清单（含优先级、所需补充文档类型）
- 统计汇总
